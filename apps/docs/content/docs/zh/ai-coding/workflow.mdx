## 必须阅读

修改文件前，AI coding agent 必须阅读：

- 根目录 `AGENTS.md`。
- 相关 app 或 package 的 `AGENTS.md`。
- 涉及 UI 时阅读 `apps/web/DESIGN.md`。
- 修改 package exports 或边界时阅读 package README。
- 修改已记录行为时阅读最近的 docs 页面。

Agent 还应阅读计划修改的文件，以及定义现有模式的相邻测试或文档。

## 规则文件的作用

- 根目录 `AGENTS.md` 定义仓库身份、package boundaries、安全规则、CI 要求、env 规则和上游署名要求。
- App 级 `AGENTS.md` 定义该 app 拥有什么，以及禁止导入什么。
- Package 级 `AGENTS.md` 定义 package 职责、禁止改动、依赖边界和聚焦验证命令。
- `apps/web/DESIGN.md` 定义 LaunchBase 工程极简视觉系统，用于 UI 和资产工作。

这些文件不是建议，而是人和 AI 修改代码时都应遵守的本地契约。

## 推荐流程

1. 阅读相关文件并识别现有模式。
2. 对非简单改动提出最小计划。
3. 只编辑能解决任务的最小范围。
4. 运行相关验证命令。
5. 总结修改文件、验证结果和剩余风险。

## 防止架构漂移的边界

- Apps 组合 packages；packages 不能从 `apps/*` 导入。
- `@repo/ui` 保持 presentation-focused，不能导入 auth、db、rpc、telemetry、security、analytics 或 app code。
- Database schema 和 clients 保留在 `@repo/db`。
- Auth setup 保留在 `@repo/auth`。
- Redis clients 保留在 `@repo/redis`。
- Browser analytics 保留在 `@repo/analytics`；Sentry 和 Evlog 保留在 `@repo/telemetry`。
- Security headers 和 CSP 保留在 `@repo/security`。

当请求跨越边界时，说明原因，并更新相关 package README 或 docs 页面。

## 禁止事项

- 不要提交 secrets、tokens、DSNs、API keys 或 `.env` 文件。
- 不要绕过、删除或削弱 CI、lint、typecheck 或 build 检查。
- 不要修改无关架构或 package ownership。
- 不要隐藏 LaunchBase 派生自上游 `stack-found/vazen` 的事实。
- 不要把 `apps/web/src/proxy.ts` 当成 admin 权限边界。

## Review 工作流

AI-assisted 任务交付时使用这个结构：

1. 修改了哪些文件以及原因。
2. 验证命令和结果。
3. 是否触及旧品牌或安全敏感内容。
4. 已知限制，或无法运行的检查。
5. 是否更新 docs、env examples、package READMEs 或 design rules。

## Codex prompt 示例

```txt
You are working in hexuntao/launchbase. Before editing, read root AGENTS.md,
the relevant app/package AGENTS.md, and nearby code. Make the smallest change
that solves the task, preserve Vazen attribution, do not commit secrets, and
run pnpm lint, pnpm typecheck, and pnpm build before summarizing results.
```
