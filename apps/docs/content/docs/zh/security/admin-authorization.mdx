## Proxy 不是权限边界

`apps/web/src/proxy.ts` 只用于改善未登录用户访问 admin 路由时的跳转体验。它检查的是 session cookie，不是当前用户角色，也不是数据库中的真实 session 状态。

不要把 proxy redirect 当成权限校验。真正的权限边界必须放在能读取 session 并校验当前角色的服务端代码中。

Admin-only 行为应放在这些边界：

- Server component。
- Route handler。
- Server action。
- 使用 `adminProcedure` 的 RPC procedure。

## RPC 示例

Admin-only RPC handler 使用 `adminProcedure`：

```ts
import { adminProcedure } from "../orpc";

export const appRouter = {
  adminStats: adminProcedure.handler(({ context }) => {
    return {
      userId: context.user.id,
      role: context.user.role,
    };
  }),
};
```

`adminProcedure` 会在没有 session 时返回 `UNAUTHORIZED`，在用户不是 admin 时返回 `FORBIDDEN`。Admin 读写操作应放在该 procedure 或等价的服务端校验之后。
