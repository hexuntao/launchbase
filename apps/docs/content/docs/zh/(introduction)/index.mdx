LaunchBase 是一个生产级 TypeScript monorepo 启动模板，用于使用 Turborepo、Next.js、PostgreSQL 和 Vercel 构建现代全栈产品。

它面向需要清晰基础设施的开发者：类型化包边界、可部署默认配置、数据库访问、可观测性入口和明确的 AI 编码规则。英文文档作为开源社区默认入口，简体中文文档作为完整的本地开发者镜像。

## 包含内容

- `apps/web`：主要的 Next.js 产品应用。
- `apps/docs`：当前 Fumadocs 文档站点。
- `packages/ui`：共享 UI 原语、全局样式和字体。
- `packages/auth`：Better Auth 服务端与客户端配置。
- `packages/db`：Drizzle schema、PostgreSQL 客户端和迁移配置。
- `packages/redis`：Upstash Redis 客户端、key 前缀和限流工具。
- `packages/rpc`：oRPC context、procedure 和 router 导出。
- `packages/security`：共享安全响应头与 CSP 配置。
- `packages/analytics`：PostHog 浏览器分析。
- `packages/telemetry`：Sentry 和 Evlog 集成。
- `packages/email`：React Email 渲染工具和模板。

## 为什么适合 AI 编码

LaunchBase 不依赖 prompt 记忆，而是把 agent 工作约束写入仓库。根目录和 package 级 `AGENTS.md` 定义范围、禁止事项、验证命令和包边界。`apps/web/DESIGN.md` 定义 UI 工作的视觉系统。文档页同步这些规则，让人和 agent 使用同一份契约。

## Monorepo 结构

应用负责组合共享包。共享包不能从 `apps/*` 导入代码。数据库访问属于 `@repo/db`，认证属于 `@repo/auth`，路由处理器在 `apps/web` 中保持轻量，共享 UI 保持在 `@repo/ui` 中。

## 部署和验证

任何代码改动都应保持以下检查通过：

```bash
pnpm lint
pnpm typecheck
pnpm build
```

在单一区域开发时先运行聚焦检查，合并前再运行完整基线。

使用 Vercel 部署 `apps/web`：Node.js 22，install command 为 `pnpm install --frozen-lockfile`，build command 为 `pnpm web:build`，环境变量见部署指南。
