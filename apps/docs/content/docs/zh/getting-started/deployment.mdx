## Fork 和 clone

先 fork `hexuntao/launchbase`，再 clone 你的 fork：

```bash
git clone https://github.com/<your-org>/<your-repo>.git
cd <your-repo>
pnpm install
```

## 配置 env

复制示例文件：

```bash
cp .env.example .env
cp apps/web/.env.example apps/web/.env
```

生成 Better Auth secret，并写入本地 env 文件：

```bash
openssl rand -base64 32
```

本地必填值：

```txt
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/launchbase_db"
UPSTASH_REDIS_REST_URL="http://localhost:8079"
UPSTASH_REDIS_REST_TOKEN="launchbase"
BETTER_AUTH_SECRET="replace-with-at-least-32-random-characters"
```

## 本地运行

启动 PostgreSQL、Redis 和 Redis HTTP bridge：

```bash
pnpm docker:up
```

运行 web app：

```bash
pnpm web:dev
```

运行 docs app：

```bash
pnpm docs:dev
```

web app 使用 `web.launchbase.localhost`；docs 使用 `docs.launchbase.localhost`。

## 验证

运行与 CI 相同的检查：

```bash
pnpm lint
pnpm typecheck
pnpm build
```

仅修改 docs 时，也运行：

```bash
pnpm docs:build
```

## 部署到 Vercel

为 `apps/web` 创建 Vercel 项目。使用 Node.js 22 和 pnpm。monorepo 构建通过 pnpm 和 Turborepo 使用 workspace packages。

设置以下 Vercel 必填环境变量：

- `DATABASE_URL`
- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`
- `BETTER_AUTH_SECRET`

可选集成：

- Google 登录：设置 `GOOGLE_CLIENT_ID` 和 `GOOGLE_CLIENT_SECRET`。
- Sentry：设置 `NEXT_PUBLIC_SENTRY_DSN`、`NEXT_PUBLIC_SENTRY_CSP_REPORT_ENDPOINT`、`SENTRY_ORG`、`SENTRY_PROJECT` 和 `SENTRY_AUTH_TOKEN`。
- PostHog：设置 `NEXT_PUBLIC_POSTHOG_KEY` 和 `NEXT_PUBLIC_POSTHOG_HOST`。

未启用集成时保持可选值为空。不要提交真实 token 或 DSN。
