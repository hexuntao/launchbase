## Required reading

Before changing files, an AI coding agent must read:

- Root `AGENTS.md`.
- The relevant app or package `AGENTS.md`.
- `apps/web/DESIGN.md` for UI work.

The agent should also read the files it plans to edit and nearby tests or docs that define the existing pattern.

## Recommended workflow

1. Read relevant files and identify existing patterns.
2. Propose a minimal plan for non-trivial changes.
3. Edit the smallest scope that solves the task.
4. Run the relevant validation commands.
5. Summarize changed files, validation results, and remaining risks.

## Forbidden actions

- Do not commit secrets, tokens, DSNs, API keys, or `.env` files.
- Do not bypass, remove, or weaken CI, lint, typecheck, or build checks.
- Do not change unrelated architecture or package ownership.
- Do not hide that LaunchBase is derived from upstream `stack-found/vazen`.
- Do not treat `apps/web/src/proxy.ts` as an admin authorization boundary.
