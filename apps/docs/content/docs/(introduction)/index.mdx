LaunchBase is a production-grade TypeScript monorepo starter for building modern full-stack products with Turborepo, Next.js, PostgreSQL, and Vercel.

It is for developers who want a clean foundation with typed package boundaries, deployment-ready defaults, database access, observability hooks, and explicit AI coding rules. The docs use English as the default open-source entry and Simplified Chinese as a complete local developer mirror.

## What is included

- `apps/web`: the primary Next.js product app.
- `apps/docs`: this Fumadocs documentation site.
- `packages/ui`: shared UI primitives, global styles, and fonts.
- `packages/auth`: Better Auth server and client configuration.
- `packages/db`: Drizzle schema, PostgreSQL client, and migration config.
- `packages/redis`: Upstash Redis client, key prefixes, and rate limiting.
- `packages/rpc`: oRPC context, procedures, and router exports.
- `packages/security`: shared security headers and CSP configuration.
- `packages/analytics`: PostHog browser analytics.
- `packages/telemetry`: Sentry and Evlog integration.
- `packages/email`: React Email rendering utilities and templates.

## Why it is AI-ready

LaunchBase makes agent work explicit instead of relying on prompt memory. Root and package-level `AGENTS.md` files define scope, forbidden actions, validation, and package boundaries. `apps/web/DESIGN.md` defines the visual system for UI work. Docs pages mirror these rules so humans and agents can work from the same contract.

## Monorepo structure

Apps compose shared packages. Shared packages do not import from `apps/*`. Database access belongs in `@repo/db`, authentication belongs in `@repo/auth`, route handlers stay thin in `apps/web`, and shared UI remains presentation-focused in `@repo/ui`.

## Deploy and validate

Every code change should keep these checks passing:

```bash
pnpm lint
pnpm typecheck
pnpm build
```

Run focused checks when working in one area, then run the full baseline before merging.

Deploy `apps/web` on Vercel with `pnpm install --frozen-lockfile`, `pnpm web:build`, Node.js 22, and the environment variables listed in the deployment guide.
