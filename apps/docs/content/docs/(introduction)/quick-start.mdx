## Prerequisites

- Node.js 22 or newer.
- pnpm 10.33.4 or compatible pnpm 10 release.
- Docker for local PostgreSQL, Redis, and the Redis HTTP bridge.

## Install

```bash
pnpm install
cp .env.example .env
cp apps/web/.env.example apps/web/.env
```

Generate a local Better Auth secret with at least 32 characters:

```bash
openssl rand -base64 32
```

Set the generated value as `BETTER_AUTH_SECRET` in the copied env files. Do not commit `.env` files or real secrets.

## Start services

```bash
pnpm docker:up
```

This starts PostgreSQL on `localhost:5432`, Redis on `localhost:6379`, and the local Redis HTTP bridge on `localhost:8079`.

Stop services when finished:

```bash
pnpm docker:down
```

## Run apps

Run the web app:

```bash
pnpm web:dev
```

Run the docs app:

```bash
pnpm docs:dev
```

The dev scripts use `portless`. The default local hostnames are `web.launchbase.localhost` and `docs.launchbase.localhost`.

## Build and validate

```bash
pnpm lint
pnpm typecheck
pnpm build
```

Focused checks for documentation work:

```bash
pnpm --filter docs typecheck
pnpm docs:build
```

Focused checks for web work:

```bash
pnpm --filter web typecheck
pnpm web:build
```

## Common troubleshooting

- `BETTER_AUTH_SECRET` must be at least 32 characters.
- `DATABASE_URL` must point to the local PostgreSQL container or a reachable PostgreSQL database.
- `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` are required because Redis env validation is shared by auth and database packages.
- If `web.launchbase.localhost` or `docs.launchbase.localhost` does not resolve in your browser, use the local port printed by the dev server.
- If a build fails after changing MDX navigation, run `pnpm --filter docs typecheck` to regenerate Fumadocs source files and Next.js typegen.
