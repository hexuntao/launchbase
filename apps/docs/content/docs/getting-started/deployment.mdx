## Fork and clone

Fork `hexuntao/launchbase`, then clone your fork:

```bash
git clone https://github.com/<your-org>/<your-repo>.git
cd <your-repo>
pnpm install
```

## Configure env

Copy the example files:

```bash
cp .env.example .env
cp apps/web/.env.example apps/web/.env
```

Generate a Better Auth secret and set it in your local env files:

```bash
openssl rand -base64 32
```

Required local values:

```txt
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/launchbase_db"
UPSTASH_REDIS_REST_URL="http://localhost:8079"
UPSTASH_REDIS_REST_TOKEN="launchbase"
BETTER_AUTH_SECRET="replace-with-at-least-32-random-characters"
```

## Run locally

Start PostgreSQL, Redis, and the Redis HTTP bridge:

```bash
pnpm docker:up
```

Run the web app:

```bash
pnpm web:dev
```

Run the docs app:

```bash
pnpm docs:dev
```

The web app uses `web.launchbase.localhost`; docs use `docs.launchbase.localhost`.

## Validate

Run the same checks used by CI:

```bash
pnpm lint
pnpm typecheck
pnpm build
```

For docs-only changes, also run:

```bash
pnpm docs:build
```

## Deploy to Vercel

Create a Vercel project for `apps/web`. Use Node.js 22 and pnpm. The monorepo build uses workspace packages through pnpm and Turborepo.

Set these required Vercel environment variables:

- `DATABASE_URL`
- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`
- `BETTER_AUTH_SECRET`

Optional integrations:

- Google sign-in: set `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`.
- Sentry: set `NEXT_PUBLIC_SENTRY_DSN`, `NEXT_PUBLIC_SENTRY_CSP_REPORT_ENDPOINT`, `SENTRY_ORG`, `SENTRY_PROJECT`, and `SENTRY_AUTH_TOKEN`.
- PostHog: set `NEXT_PUBLIC_POSTHOG_KEY` and `NEXT_PUBLIC_POSTHOG_HOST`.

Keep optional values empty when the integration is not enabled. Never commit real tokens or DSNs.
